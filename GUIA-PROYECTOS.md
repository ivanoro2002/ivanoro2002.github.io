# Guia para cargar proyectos

## Como conviene subir un proyecto

El portafolio no guarda el proyecto completo adentro de la web. Guarda una ficha profesional del proyecto.

Lo recomendado es:

1. Subir el codigo del proyecto a GitHub.
2. Crear o mejorar el `README.md` del proyecto.
3. Si el proyecto tiene demo, publicarla en Netlify, Vercel, GitHub Pages u otra opcion gratuita.
4. Entrar al admin del portafolio.
5. Agregar una ficha en `Proyectos`.
6. Pegar el link del repositorio y, si existe, el link de la demo.

## Que campos completar

- Nombre: titulo claro del proyecto.
- Estado: en desarrollo, terminado, universitario, practica o activo.
- Tipo: frontend, backend, escritorio, base de datos, etc.
- Descripcion: que hace el proyecto.
- Problema u objetivo: que querias resolver o practicar.
- Aprendizaje: que aprendiste al hacerlo.
- Proximo paso: que mejora falta.
- Tecnologias: HTML, CSS, JavaScript, Java, Python, SQL, Git, etc.
- Enlaces: repositorio y demo.
- Imagen: opcional, una captura liviana del proyecto.

## Si solo tengo el repositorio

Esta bien. Para un perfil inicial IT, un repositorio bien presentado ya sirve.

Lo ideal es que el repositorio tenga:

- `README.md`
- instrucciones para ejecutar
- tecnologias usadas
- capturas si corresponde
- una explicacion breve del objetivo

## Si tengo archivos sueltos

Conviene no subir archivos sueltos directamente al portafolio. Primero armamos un repositorio de GitHub para ese proyecto y despues agregamos el link en la ficha del portafolio.

## Imagenes

Las imagenes son opcionales. Si subis una captura, mejor que sea liviana para que la web no se vuelva pesada.
